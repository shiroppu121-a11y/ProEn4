# ProEn4

# 横スクロールアクション2Dゲーム

## 1. 概要

本作品は、プレイヤーが横スクロールするステージを進み、敵や障害物を避けながらゴールを目指す2Dアクションゲームである。

プレイヤーはキーボードを使用して移動・ジャンプを行い、ステージ上に配置されたアイテムを取得して使用することができる。アイテムには、移動速度を上昇させる「スピードアップ」、一定時間敵との接触によるゲームオーバーを防ぐ「無敵」、敵に向かって弾を発射して倒す「投擲」の3種類がある。

また、ゲーム開始からゴール到達までの時間を計測し、クリアタイムを記録する機能を実装している。

---

## 2. ゲームの目的

プレイヤーを操作してステージを進み、敵との衝突を避けながらゴールに到達することを目的とする。

ゴールに到達するとゲームクリアとなり、ゲーム開始からゴール到達までのクリアタイムが表示される。また、クリアタイムは保存され、タイトル画面から過去の記録を確認することができる。

---

## 3. 基本操作

| キー  | 操作                     |
| --- | ---------------------- |
| A   | 左方向へ移動                 |
| D   | 右方向へ移動                 |
| W   | ジャンプ                   |
| E   | 所持しているアイテムを使用          |
| Esc | ポーズ画面の表示・解除            |
| R   | ゲームオーバー・クリア後にリスタート     |
| T   | ゲームオーバー・クリア後にタイトル画面へ戻る |

---

## 4. ゲームの流れ

### 4.1 タイトル画面

ゲーム起動時にはタイトル画面が表示される。

タイトル画面では以下の操作を行うことができる。

* Enterキー：ゲーム開始
* Hキー：過去のクリアタイムを表示

### 4.2 ゲーム開始

ゲーム開始を選択するとゲーム画面へ移行し、クリアタイムの計測を開始する。

プレイヤーはAキーとDキーで左右に移動し、Wキーでジャンプする。

### 4.3 アイテムの取得

ステージ上には複数のアイテムが配置されている。

プレイヤーがアイテムに接触すると、そのアイテムを取得することができる。

ただし、プレイヤーがすでにアイテムを所持している場合、新しいアイテムを取得することはできない。

取得したアイテムは画面上に表示され、Eキーを押すことで使用する。

### 4.4 アイテムの種類

#### スピードアップ

使用するとプレイヤーの移動速度が上昇する。

#### 無敵

使用すると5秒間、敵に接触してもゲームオーバーにならない。

#### 投擲

使用するとプレイヤーの前方に弾を発射する。

発射された弾が敵に接触すると、その敵を倒すことができる。

### 4.5 敵との接触

プレイヤーが敵に接触するとゲームオーバーとなる。

ただし、無敵アイテムを使用している間は敵に接触してもゲームオーバーにならない。

### 4.6 ゴール

プレイヤーがゴールオブジェクトに接触するとゲームクリアとなる。

ゴールに到達した時点でタイマーを停止し、クリアタイムを表示する。

クリアタイムは保存され、後からタイトル画面の記録画面で確認できる。

---

## 5. ポーズ機能

ゲーム中にEscキーを押すとゲームが一時停止し、ポーズ画面が表示される。

ポーズ中はプレイヤーや敵の動作、タイマーの進行が停止する。

ポーズ画面では以下の操作を行うことができる。

* Escキー：ゲームを再開
* Tキー：タイトル画面に戻る

ポーズ中にタイトル画面へ戻った場合、そのゲームの進行状況やクリアタイムは保存されない。

---

## 6. クリアタイム記録機能

ゲームクリア時にクリアタイムを保存する。

保存されたクリアタイムはタイトル画面から確認することができ、過去の記録を短い順に表示する。

過去に一度もゲームをクリアしていない場合は、「No clear records yet」と表示する。

---

## 7. システム構成

本ゲームは、以下の主要なクラスおよびモジュールによって構成されている。

### Playerクラス

プレイヤーの位置、移動速度、ジャンプ、重力、アイテムの所持・使用などを管理する。

### Enemyクラス

敵の位置や移動範囲、敵の種類などを管理する。

### Itemクラス

アイテムの位置、種類、取得状態などを管理する。

### Bulletクラス

投擲アイテムによって発射される弾の位置や移動、敵との衝突判定を管理する。

### Mapクラス

ステージ上の足場やブロックを管理する。

### update.py

プレイヤーや敵の更新処理、カメラの移動、当たり判定などを行う。

### draw.py

ゲーム画面、プレイヤー、敵、アイテム、ゴール、UI、タイトル画面、ポーズ画面などの描画を行う。

### main.py

ゲーム全体の処理を管理し、タイトル画面・ゲーム画面・記録画面などのシーン切り替えを行う。

## 11. まとめ

本作品では、PythonとPygameを用いて横スクロール型の2Dアクションゲームを制作した。

プレイヤーの移動やジャンプ、敵との衝突判定、アイテムの取得・使用、敵への攻撃、ゴール判定など、2Dアクションゲームの基本的な要素を実装した。

さらに、クリアタイムの計測と保存機能を実装することで、プレイヤーがより短い時間でのクリアを目指せるようにした。

ゲームの各機能を複数のクラスやモジュールに分割することで、プレイヤー、敵、アイテム、ステージ、ゲーム更新、画面描画などの役割を分離し、機能ごとに管理しやすい構成とした。
